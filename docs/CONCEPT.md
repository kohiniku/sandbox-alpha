# コンセプト詳細

## 背景と問題意識

### 従来の投資戦略開発の課題

1. **人手依存**: 仮説立案→実装→検証の全工程が人間の手作業
2. **環境構築の重複**: 各戦略ごとにバックテスト環境を再構築
3. **失敗の散逸**: 不採用戦略の教訓が記録されず、同じ失敗を繰り返す
4. **探索空間の制約**: 人間が思いつく戦略・パラメータに限界

### なぜ自律的PDCAが必要か

投資戦略の「アルファ（超過収益）」は：
- **一時的**: 市場が適応するため、有効期間は限定的
- **非自明**: 単純なルールでは発見困難
- **多様**: 異なる市場環境・銘柄・期間で有効性が異なる

→ 大量の仮説を高速にテストし、成功パターンを蓄積する必要がある

### 各フェーズの詳細

#### 1. 仮説生成 (Hypothesis Generation)

**現在の手法**: ランダムサンプリング
- 戦略テンプレート（SMA Crossover, Mean Reversion, Momentum）から選択
- 銘柄プール（AAPL, MSFT, NVDA, BTC-USD等）から選択
- パラメータ空間からランダムに値をサンプリング

**将来の拡張**:
- LLMベースの仮説生成（ナレッジベースを参照して賢い仮説を生成）
- 遺伝的アルゴリズム（成功戦略の変異・交叉）
- 強化学習エージェント（過去の成功・失敗から学習）

#### 2. サンドボックス構築 (Sandbox Construction)

**現在の手法**: Python venv隔離環境
- 各イテレーションごとに独立したsubprocessを起動
- 依存関係の衝突を防止
- タイムアウト設定（120秒）

**将来の拡張**:
- Docker使い捨てコンテナ（完全隔離）
- Kubernetes Job（並列実行）
- クラウドファンクション（スケーラブル）

#### 3. ペーパートレード (Paper Trading)

**現在の手法**: 過去データでのバックテスト
- yfinanceから2年分のOHLCVデータを取得
- シグナル生成→ポジション計算→リターン集計
- 取引コストはゼロ（将来追加予定）

**将来の拡張**:
- リアルタイムペーパートレード（実際の市場データで数日間実行）
- Polymarket API統合（予測市場でのベッティング）
- スリッページ・手数料モデルの追加

#### 4. パフォーマンス評価 (Performance Evaluation)

**現在のメトリクス**:
- Sharpe Ratio（リスク調整後リターン）
- Total Return（累積リターン）
- Max Drawdown（最大ドローダウン）

**閾値**:
- Sharpe ≥ 0.3
- Return ≥ 5%
- Drawdown ≥ -25%

**将来の拡張**:
- Sortino Ratio（下方向リスク考慮）
- Calmar Ratio（ドローダウン考慮）
- 統計的有意性検定（t検定、ブートストラップ）
- ベンチマーク比較（SPY Buy & Hold等）

#### 5. ナレッジ蓄積 (Knowledge Accumulation)

**現在の形式**: JSONファイル
- テスト済み仮説IDのリスト
- 採用戦略の詳細レコード
- 不採用戦略の詳細レコード
- イテレーション数

**将来の拡張**:
- ベクトルDB（成功パターンのセマンティック検索）
- グラフDB（戦略間の関係性）
- 自動ドキュメント生成（成功要因の分析）

## なぜ「使い捨て」が重要か

### 1. 過学習防止

同じ環境で繰り返しテストすると：
- 戦略が特定の市場条件に過剰適合
- out-of-sampleでの性能が劣化

→ 毎回異なる環境（期間・銘柄・パラメータ）でテスト

### 2. 多様性確保

固定環境では：
- 限られた市場条件のみをテスト
- ブラックボックス的な失敗モードを見逃す

→ ランダムサンプリングで多様な条件を探索

### 3. コスト効率

常時環境を保持すると：
- クラウドリソースの無駄
- メンテナンスコスト

→ 必要な時だけ起動、不要になったら破棄

## 学術的意義

### 研究ギャップ

既存研究（2024-2026年）の大半は：
- **事前定義された環境**でエージェントが戦略を学習
- エージェントが**環境自体を設計・修正**するメタレベルの自律性は未開発

→ 本研究は「環境の自律的構築」に焦点

### 関連分野

- **AutoML**: ハイパーパラメータ探索の自動化
- **Neural Architecture Search (NAS)**: ネットワーク構造の自動設計
- **AI Scientist**: 科学的研究の完全自動化
- **Voyager**: 自律的スキル獲得・蓄積

→ これらの手法を投資戦略開発に応用

## 実用上の意義

### 個人投資家への恩恵

- 専門知識がなくても多様な戦略を探索可能
- 24/7自律的に探索を継続
- 失敗から学習して探索効率を改善

### 機関投資家への応用

- クオンツチームの補助ツール
- 新しいアルファシグナルの自動発掘
- ストラテジーライブラリの自動拡充

## 倫理的考慮事項

### リスク

- 自律的取引システムの暴走
- 市場への悪影響（流動性枯渇、フラッシュクラッシュ）
- 規制回避の可能性

### 対策

- **段階的導入**: ペーパートレード→少額実資金→本格運用
- **ガードレール**: 最大ポジションサイズ、1日取引回数制限
- **監査ログ**: 全意思決定の記録・説明可能性
- **人間監視**: 定期的なパフォーマンスレビュー

---

## まとめ

本プロジェクトは：

1. **使い捨てサンドボックス**で過学習を防止
2. **自律的PDCAループ**で大量の仮説を高速テスト
3. **ナレッジ蓄積**で探索効率を継続改善

→ アルファ戦略の自動発見を実現

**最終目標**: AIエージェントが人間研究者と協調して、新しい投資戦略を継続的に発見・改善するシステム
